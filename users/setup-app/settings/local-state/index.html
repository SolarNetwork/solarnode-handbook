    13. Local State¶

    SolarNode provides a persistent "state" database for use by plugins and expressions. The Settings > Local State page lets you manage this database. The term "local" refers to the fact that the persisted state database is specific (local) to each node, and not published to the SolarNetwork cloud.

    See the Local State information page for general information about Local State entities.

    13.1 Managing Local State¶

    Click the + Create Local State button to create a new local state entity, or click on the Key value of any existing state entity to edit it. An edit dialog will be shown where you can enter the details.

    To delete a Local State entity, click on its link to open the edit dialog, then click the Delete button.

    13.2 CSV Backups¶

    The Local State data is automatically backed up on a configurable schedule. You can adjust the schedule under Settings > Services > Local State Local Backup. Use the CSV Backup & Restore form to download the current Local State database, or one of the automatic backups, as a CSV file.
